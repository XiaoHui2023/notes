# ReAct

**ReAct = Reason + Act**。

一轮对话只回答一次：用户提出问题，ai 给出回答，这轮就结束了。

ReAct 不只回答一次。ai 会一边思考，一边使用工具；工具返回结果后，再判断下一步怎么做，直到任务完成。

## lint

比如写代码时，ai 先修改源码，然后检查 linter。

如果 linter 不通过，ai 会读报错，回到源码里重新修改，再次检查，直到结果合格。

## agent 模式

ReAct 就是 agent 模式。

平时的 chat 模式，更像跟 ai 进行一轮对话。用户问一次，ai 答一次。

agent 模式会反复执行：思考、调用工具、读取结果、继续判断下一步。反复的次数多了，就会消耗很多 token。

所以可以把贵的 ai 用在对话和判断上，把便宜的 ai 用在 agent 模式里跑任务。

如果一段对话已经很长，继续聊天也会带上很长的上下文。上下文越长，新增信息的收益越低，token 消耗却会越来越高。

如果后续任务不依赖前面的上下文，可以关掉当前对话，重新开一个新的对话。让每次聊天的上下文保持短一些，也是一种节约 token 的方法。

## Debug 模式

Debug 模式更像一个会自己动手排查问题的过程。

ai 会自己运行命令，读取运行结果，再根据报错、日志或测试输出判断问题在哪里。下一步可能是继续运行，也可能是回到代码里修改。
